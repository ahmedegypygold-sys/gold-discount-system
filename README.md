# نظام حساب الخصم المطور

المجلد يحتوي على نسختين:

- `app.py`: واجهة Streamlit.
- `discount_system.html`: نسخة HTML مستقلة تعمل من المتصفح مباشرة، وتصدر HTML / Excel / PDF.
- `discount_rules.py`: قواعد الحساب المستخدمة في Streamlit.
- `new_egypt_gold_logo.jpeg`: لوجو الشركة المستخدم في الواجهة والتقارير.

## التشغيل على Streamlit

```powershell
cd "C:\Users\a.younis\Documents\New project\discount_system_pro"
pip install -r requirements.txt
streamlit run app.py
```

## تشغيل نسخة HTML

افتح الملف التالي في المتصفح:

```text
C:\Users\a.younis\Documents\New project\discount_system_pro\discount_system.html
```

أو شغلها كلينك داخلي من نفس المجلد باستخدام السيرفر المحلي الخاص بالنظام:

```powershell
python offline_server.py
```

ثم افتح:

```text
http://127.0.0.1:8765/discount_system.html
```

ولو المستخدمون على نفس شبكة الواي فاي، استخدم IP الجهاز بدل `127.0.0.1`.

## التشغيل أوفلاين على شبكة الشركة

1. افتح الملف `start_offline_server.bat`.
2. سيظهر رابط جهازك:
   - على نفس الجهاز: `http://127.0.0.1:8765/discount_system.html`
   - للناس على نفس شبكة الواي فاي/الشركة: `http://192.168.1.10:8765/discount_system.html`
3. اترك نافذة التشغيل مفتوحة طول ما الناس تستخدم النظام.
4. لو قفلت النافذة، اللينك سيقف.
5. لو IP الجهاز اتغير، شغل `ipconfig` وخذ IPv4 Address الجديد وضعه بدل `192.168.1.10`.

ملاحظة: الإيميل الخاص بطلب الصلاحية يحتاج برنامج إيميل أو إنترنت لإرسال الرسالة. النظام نفسه يشتغل محلياً على الشبكة.

مهم: ملف التشغيل يستخدم `offline_server.py` حتى يحفظ المستخدمين مركزياً في `users.json`. لا تشغل `python -m http.server` لهذا النظام، لأنه لا يحفظ الصلاحيات لكل المتصفحات.

## بيانات الدخول

- الأدمن الافتراضي:

  - كلمة المرور: `Eg@2026`
- الأدمن فقط يقدر يرفع/يثبت ملف المبيعات ويضيف مستخدمين.
- المستخدم العادي يدخل ويرفع ملف السداد فقط، ثم يحسب الخصم من قاعدة المبيعات المثبتة.
- المستخدمون وقاعدة المبيعات محفوظون في نفس المتصفح عبر `localStorage`.
- شاشة الدخول فيها زر `طلب صلاحية دخول` يفتح رسالة إيميل جاهزة للأدمن باسم طالب الدخول وبياناته.
- طلبات الصلاحية تذهب إلى: `AHMED.YOUNIS@NEWEGYGOLD.COM`.

## طريقة الموافقة على المستخدمين

1. المستخدم يفتح اللينك ويضغط `طلب صلاحية دخول`.
2. يكتب اسمه وبيانات التواصل وسبب الدخول.
3. يرسل الإيميل الجاهز إلى `AHMED.YOUNIS@NEWEGYGOLD.COM`.
4. بعد الموافقة، افتح النظام بحساب الأدمن:
   - اسم المستخدم: `admin`
   - كلمة المرور: `Eg@2026`
5. من جزء `إضافة مستخدم` اكتب اسم المستخدم وكلمة المرور والصلاحية، ثم اضغط `حفظ المستخدم`.
6. الحساب يتم حفظه مركزياً في `users.json`.
7. ابعت للمستخدم اسم المستخدم وكلمة المرور فقط، ويقدر يدخل من أي متصفح أو جهاز على نفس الشبكة.
8. لينك التفعيل سيظل ظاهر كاختيار احتياطي، لكنه لم يعد مطلوباً طالما السيرفر يعمل من `start_offline_server.bat`.

ملاحظة مهمة: لينك التفعيل يحتوي بيانات الدخول بصيغة قابلة للقراءة لمن يملك الرابط. لا ترسل الرابط إلا للمستخدم المقصود. لو عايز صلاحيات مركزية تتفعل وتتقفل لكل الناس أونلاين من عندك فوراً، يلزم Backend بسيط أو خدمة مثل Firebase / Supabase / Google Apps Script.

## النشر على Firebase المجاني

الخطة المجانية المناسبة هي Firebase Spark:

- Firebase Hosting يعطي مساحة مجانية لمحتوى الموقع حتى `10 GB`.
- Cloud Firestore يعطي تخزين مجاني للداتا حتى `1 GiB`.
- قاعدة المبيعات الحالية حجمها حوالي `6.6 MB` فقط، يعني مناسبة جداً كبداية.

خطوات النشر:

1. افتح [Firebase Console](https://console.firebase.google.com/) واعمل مشروع جديد.
2. فعّل Firestore Database في وضع Native mode.
3. من Project settings أضف Web App وانسخ إعدادات Firebase.
4. افتح `firebase-config.js` واستبدل قيم `YOUR_FIREBASE_*` بالقيم الحقيقية.
5. بعد أول نشر، ضع رابط الموقع العام في `PUBLIC_SITE_URL`، مثال:
   ```javascript
   window.PUBLIC_SITE_URL = "https://YOUR_FIREBASE_PROJECT_ID.web.app";
   ```
   هذا مهم حتى لينك تفعيل المستخدم يفتح عند أي مستخدم، وليس فقط على جهاز الأدمن.
6. افتح `.firebaserc` واستبدل `YOUR_FIREBASE_PROJECT_ID` برقم/اسم مشروع Firebase.
7. ثبّت Firebase CLI وسجّل دخول:
   ```powershell
   npm install -g firebase-tools
   firebase login
   ```
8. انشر الموقع والقواعد:
   ```powershell
   firebase deploy --only hosting,firestore:rules
   ```
9. بعد النشر افتح الموقع بحساب الأدمن، ومن `إعدادات الجلسة` اكتب رابط الموقع العام على Firebase واحفظه. بعد ذلك أنشئ المستخدمين، وستكون لينكات التفعيل صالحة من أي مكان.

ملاحظة أمان: هذه نسخة بسيطة للموقع بحسابات داخلية محفوظة في الداتا، وليست بديل نظام صلاحيات مؤسسي كامل. لو الموقع سيُفتح خارج الشركة لعدد كبير، الأفضل لاحقاً إضافة Firebase Authentication وقواعد Firestore مغلقة.

## الملفات المطلوبة

1. ملف المبيعات `Sales By Item Category.csv`.
2. ملف السداد `Payment CSV` ويحتوي على الأقل:
   - `Reference`
   - `GoldDays`
   - `CashDays` إن وجد
3. يمكن بدلاً من ملف السداد إدخال الفواتير يدوياً داخل الموقع، كل سطر بهذا الشكل:
   - `SA25FW-01197 25 1`
   - رقم الفاتورة، أيام الذهب، ثم أيام النقدية اختياري.

## القواعد المحفوظة

- الشرائح من صافي الكمية: `SA - SR`.
- عيار 18 فقط يستخدم حدود عيار 18 الصغيرة.
- وجود عيار 21 أو الاثنين يستخدم إجمالي `G18 + G21` على حدود عيار 21.
- VIP يحصل على شريحة 8.
- فترة السماح تتفعل عندما كل فواتير SA المسددة ذهبها أقل من أو يساوي 37 يوم.
- أي فاتورة ذهبها أكبر من 75 أو نقديتها أكبر من 75 يكون خصمها صفر.
- تعجيل الدفع يضيف `+3.5` لفواتير SA عندما النقدية أقل من أو تساوي يومين، وللمرتجعات عند تفعيل التعجيل.
- استقطاعات النقدية محفوظة حسب الصنف والنطاق.
- المرتجعات SR تأتي من ملف المبيعات وتخصم بالسالب.
- الفواتير متعددة الأصناف تحسب كل صنف بسعره داخل نفس الفاتورة.
- يظهر تقرير أعمار المديونيات وتوزيع الخصم على الفروع.
